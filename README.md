# article-INoT

Репозиторий статьи о мега-агентной архитектуре с внутренней ролевой сегрегацией на основе Introspection of Thought (INoT).

В репозитории поддерживаются две рабочие версии статьи:

- `converted_article_springer.tex` — английская версия
- `converted_article_springer_ru.tex` — русская версия

## Состав репозитория

- `converted_article_springer.tex` — основной английский исходник статьи
- `converted_article_springer_ru.tex` — основной русский исходник статьи
- `sn-jnl.cls` — класс Springer Nature Journal
- `.vscode/settings.json` — локальные настройки LaTeX Workshop для сборки через MiKTeX
- `build.ps1` — основной скрипт сборки и очистки
- `build-en.ps1` — короткий совместимый запуск английской сборки
- `build-ru.ps1` — короткий совместимый запуск русской сборки
- `.gitignore` — исключения для артефактов сборки и локального шума
- `CHANGELOG.md` — журнал изменений проекта

## Принципы ведения статьи

- Английская и русская версии хранятся отдельно и поддерживаются в актуальном состоянии одновременно.
- В Git коммитятся только исходники и действительно нужные служебные файлы.
- Артефакты LaTeX-сборки в репозиторий не добавляются.
- Сгенерированные PDF используются локально для проверки, но не хранятся в Git.

## Быстрый старт

После открытия проекта в новом терминале PowerShell доступны два удобных способа:

- `ab` — собрать русскую версию
- `ab en` — собрать английскую версию
- `ab all` — собрать обе версии
- `ab clean` — удалить артефакты сборки

Если alias ещё не подхватился, просто закройте и откройте терминал заново.
Если даже после этого alias не появился, используйте локальную команду:

```powershell
.\ab.ps1
.\ab.ps1 en
.\ab.ps1 all
.\ab.ps1 clean
```

## Сборка

Проект настроен под `LaTeX Workshop` и `MiKTeX`.

### В VS Code

Это основной и самый удобный способ работы во время редактирования.

1. Откройте нужный файл:
   - `converted_article_springer_ru.tex` для русской версии
   - `converted_article_springer.tex` для английской версии
2. Сохраните файл.
3. Если включён autobuild, сборка запустится сама.
4. Если нужна ручная сборка:
   - `Ctrl+Shift+P`
   - `LaTeX Workshop: Build LaTeX project`
5. Для просмотра PDF:
   - `Ctrl+Shift+P`
   - `LaTeX Workshop: View LaTeX PDF file`
6. Для очистки временных файлов:
   - `Ctrl+Shift+P`
   - `LaTeX Workshop: Clean up auxiliary files`

Что важно понимать:

- LaTeX Workshop собирает именно тот `.tex`, который сейчас открыт как активный файл.
- Для русской и английской версии PDF создаются отдельно.
- После сборки можно кликать по строкам в `.tex` и переходить в соответствующее место PDF через SyncTeX.

## Что происходит во время билда

Когда запускается сборка, `latexmk`:

1. читает `.tex`-файл;
2. запускает `pdflatex`;
3. создаёт промежуточные служебные файлы;
4. при необходимости автоматически делает повторный проход, чтобы:
   - обновить ссылки `\ref`
   - обновить цитирования `\cite`
   - синхронизировать оглавление, подписи и закладки PDF
5. формирует итоговый `PDF`.

## Какие файлы появляются после билда

После сборки рядом с исходником появляются:

- `*.pdf` — итоговый документ, который вы открываете и проверяете
- `*.aux` — информация о ссылках, метках и цитированиях
- `*.log` — подробный лог компиляции
- `*.out` — служебные данные для гиперссылок и закладок PDF
- `*.fls` — список файлов, которые использовались при сборке
- `*.fdb_latexmk` — внутреннее состояние `latexmk`, чтобы понимать, что нужно пересобирать
- `*.synctex.gz` — связь между исходником `.tex` и PDF для переходов туда-обратно

Эти файлы нормальны. Они нужны LaTeX во время работы, но не коммитятся в Git.

## Что использовать на практике

- Для обычной работы в редакторе: `LaTeX Workshop`
- Для быстрой ручной сборки в терминале: `ab`
- Для полной пересборки обеих версий: `ab all`
- Для очистки мусора после сборки: `ab clean`

## Важные замечания

- Если в статье появятся реальные рисунки, их нужно добавлять в репозиторий отдельно.
- Пока файлов `figure1.png` ... `figure7.png` нет, в документе используются встроенные заглушки.
- Если меняется структура статьи, желательно в том же коммите обновлять обе языковые версии.
- Первая сборка после установки MiKTeX может быть дольше обычной, потому что система докачивает пакеты и шрифты.
